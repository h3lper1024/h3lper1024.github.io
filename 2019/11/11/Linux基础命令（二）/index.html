    Linux基础命令（二）

    1.echo

    ​ echo "xxx" 打印字符串

    ​ echo $PATH 打印环境变量

    2.环境变量

    • 环境变量实质上是K=V，执行K时，实际执行的是对应的V

    • 执行环境变量时，找到第一个符合条件的执行，不往后找

    • 个人环境变量优先

    • 修改环境变量后生效：

      source etc/profile

      . ~/.bashrc

      通常使用source xxx

    • 修改环境变量后不生效，则只在当前session有效

    1. 全局环境变量

      整个系统的用户都能使用的，在etc/profile

    2. 个人环境变量

      只适用于个人用户的环境变量，在~/.bash_profile，~/.bashrc

    3. 环境变量配置

      以配置JAVA_HOME为例

      • 新建Java目录

        mkdir -p usr/java

      • 上传jdk到服务器

        rz

      • 如果用的是java的.zip包，安装unzip命令

        yum install -y unzip

      • 改变文件用户

        chown -R root:root /usr/java/*

      • 解压.zip包

        unzip jdk-xxx.zip

      • 如果使用的是.tar.gz包

        tar -zxvf xxx.tar.gz

      • 打开全局环境变量文件

        vi etc/profile

      • 指定JAVA_HOME，环境变量添加JAVA_HOME

        #env
        export JAVA_HOME=/usr/java
        export PATH=$JAVA_HOME/bin:$PATH

      • 修改后执行which java

    3.alias命令

    ​ 简化命令

    • 查看系统指定的命令别名

      alias

    • 新建别名

    1. 当前session生效

      alias home='cd /home/hadoop'

    2. 当前用户生效

      • 修改个人环境变量

        vi ~/.bashrc

        alias home='cd /home/hadoop'

      • 生效

        source ~/.bashrc

        which home

    4.history命令

    • 查看历史命令

      history

    • 清除历史操作

      history -c

    • 执行指定历史命令

      !x x指历史命令编号

    5.rm命令

    • 删除文件

      rm -f xxx

    • 删除文件夹

      rm -rf xxx

    注意：当shell脚本中有

    ​ K=’XXX’

    ​ rm -rf $K/

    一定要加判断，$K是否存在，不存在时，实际执行的是rm -rf /
