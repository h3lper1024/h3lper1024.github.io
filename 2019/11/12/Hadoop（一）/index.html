    Hadoop简介及HDFS部署

    1.Hadoop简介

    ​ 狭义：

    ​ 指Apache Hadoop软件 链接

    • HDFS (Hadoop Distributed File System)：存储，集群中的角色主要有 NameNode / DataNode/SecondaryNameNode

    • YARN (Yet Another Reduce Negotiator)：资源调度，集群中的角色主要有 ResourceManager /NodeManager

    • MapReduce 计算

    • 广义：

    ​ 指以Hadoop为主的生态圈 链接

    2.参考部署文档

    ​ Cloudera:Apache Hadoop 2.6.0-cdh5.16.2

    ​ Apache:Apache Hadoop 2.10.0

    3.准备Hadoop的CDH安装包

    ​ 本次使用：Apache Hadoop 2.6.0-cdh5.16.2

    4.准备工作

    2.1.创建用户：

    useradd hadoop

    su - hadoop

    2.2.创建文件夹：

    ​ mkdir app software sourcecode log tmp data lib

    ​ app 解压的文件夹 软连接
    ​ data 数据
    ​ lib 第三方的jar
    ​ log 日志文件夹
    ​ software 压缩包
    ​ sourcecode 源代码编译
    ​ tmp 临时文件夹

    2.3.将Hadoop安装包放到software

    2.4.修正权限

    ​ chown ruoze:ruoze /home/ruoze/software/*

    2.5解压

    ​ 到app下

    2.6创建软连接

    ​ ln -s hadoop-2.6.0-cdh5.16.2 hadoop

    5.配置JAVA环境变量

    ​ 准备Java安装包，本次使用jdk-8u45-linux-x64.gz

    ​ 适用root，配置全局环境变量

    mkdir /usr/java

    cd /usr/java

    tar -zxvf jdk-8u45-linux-x64.gz

    vi /etc/profile

    source /etc/profile

    which java

    6.显性的为Hadoop配置Java环境变量

    cd /home/hadoop/app/hadoop/etc/hadoop

    vi hadoop-env.sh

    7.配置伪分布式部署

    cd /home/hadoop/app/hadoop/etc/hadoop

    • vi core-site.xml

    • vi hdfs-site.xml

    • 配置ssh

    • 配置Hadoop环境变量

      cd ~

      vi .bashrc

      source .bashec

      which hadoop

    • 格式化HDFS

      hdfs namenode -format

      成功标志： has been successfully formatted.

    • 第一次启动HDFS

      start-dfs.sh

    • 查看Java进程，是否启动

      jps

    • 配置DataNode、SecondaryNameNode以主机名启动

      • DataNode：

        cd /home/hadoop/app/hadoop/etc/hadoop

        vi slaves

      • SecondaryNameNode

        cd /home/hadoop/app/hadoop/etc/hadoop

        vi hdfs-site.xml
