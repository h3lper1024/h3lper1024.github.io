    Flume概述与部署

    1. Flume背景

      需求：将日志信息从源端放到HDFS

      解决方法：

      1.使用HDFS API

      ​ hdfs dfs -put

      ​ 存在的问题：

      ​ 1.需要搭配使用shell命令，定时放，延时性较高；

      ​ 2.耗时长

      ​ 3.没法监控

      2.框架

      ​ Flume

      ​ 官网介绍

    2. Flume概述

      1. Flume的组成

        Flume有3大组件

        1. Source(源端数据采集)：Flume提供了各种各样的Source、同时还提供了自定义的Source；
        2. Channel(临时存储聚合数据)：主要用的是Memory Channel和File Channel（生产最常用），生产中Channel的数据一定是要监控的，防止sink挂了，撑爆Channel；
        3. Sink（移动数据到目标端）：如HDFS、KAFKA、DB以及自定义的Sink。

      2. Flume配置

        Source 从哪收集

        avro (序列化)
        exec (命令行)
        spooling 目录
        taildir （重点）
        kafka

        Channel：数据存哪里

        memory
        kafka
        file

        Sink：数据输出到哪里

        HDFSHadoop
        logger 控制台
        avro
        kafka
